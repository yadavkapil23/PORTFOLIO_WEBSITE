NovelNet

A full-stack book discovery and review platform — search millions of books via the Google Books API, write reviews, manage your reading list, and discover what others are reading.

The Idea

NovelNet was born from a simple question: why is there no good, clean place to discover books and track what you've read? Goodreads exists, but it's slow and cluttered. I wanted to build something minimal and fast that still had a rich catalog.

The solution was to use the Google Books API as the data backbone — giving instant access to over 40 million books without maintaining a database of book metadata myself.

Architecture Overview

The application follows a standard Django MVT (Model-View-Template) pattern. The key architectural decision was the hybrid data model: book metadata is fetched from Google Books API and cached locally only when a user interacts with a book (reviews it, saves it, or searches for it).

Google Books API Integration

Search queries are sent to the Google Books Volumes endpoint with the user's query. Results are parsed and normalized into a consistent format before being displayed or stored. API responses are cached in Redis for 24 hours to minimize API calls and improve response speed.

User Authentication

Django's built-in authentication system handles user registration, login, and session management. Passwords are hashed with PBKDF2 by default. I extended the default User model with a profile that tracks reading lists and review history.

Review System

Each review is linked to a user and a book via foreign keys. Reviews include a 1–5 star rating and freeform text. The average rating for each book is computed as an aggregate query — efficient even with many reviews per book.

Supabase (PostgreSQL) Backend

I chose Supabase over SQLite for its managed PostgreSQL hosting, which gives production-grade reliability without server management. The connection uses Django's standard django.db.backends.postgresql engine with environment variables for credentials.

Deployment on Render

Render's free tier supports Python web services with automatic deploys from GitHub. The build command installs dependencies and runs Django migrations. Static files are served via WhiteNoise middleware — no separate static file server needed.

Challenges & Solutions

The main challenge was API rate limiting. The Google Books API allows 1000 free requests per day without a key and much more with one. Aggressive caching (Redis + local database) keeps actual API calls well below the limit even under heavy use.

A secondary challenge was cold starts on Render's free tier — the service sleeps after inactivity. I added a loading state to the frontend so users see feedback during the initial wake-up period rather than a blank page.
