From Script to Cloud Service

URLZAP started as a simple Python command-line tool built with argparse. You'd run python urlzap.py shorten https://my-long-url.com and get a short link back. It was useful locally, but not shareable. Turning it into a cloud-deployed web service was the real engineering challenge.

The CLI Layer

The original CLI was built with Python's argparse module — clean, dependency-free, and easy to use. Commands include shortening a URL, looking up the original URL for a short code, and listing all stored mappings.

python urlzap.py shorten https://example.com/very/long/url
# → https://urlzap.app/xK2mP

python urlzap.py expand xK2mP
# → https://example.com/very/long/url

The Web Service Layer

I wrapped the same core logic in a Flask web application, exposing it over HTTP so it could be used without the CLI. The /shorten endpoint accepts a POST request with a URL and returns the short code. The /<code> endpoint performs the redirect.

Short Code Generation

Short codes are generated by hashing the original URL with SHA-256 and taking the first 6 characters of the hex digest. This is deterministic — the same URL always produces the same short code — and has a collision probability low enough for a personal-scale tool.

import hashlib

def generate_short_code(url: str) -> str:
    return hashlib.sha256(url.encode()).hexdigest()[:6]

Containerization with Docker

The Flask application is packaged in a minimal Docker image based on python:3.11-slim. Using the slim variant keeps the image under 150MB. Gunicorn serves the application in production — Flask's development server is not suitable for real traffic.

Azure App Service Deployment

Azure App Service hosts the Docker container. The key configuration: setting WEBSITES_PORT=8000 so Azure routes incoming traffic to the correct port inside the container. The service is deployed to the Central India region.

GitHub Actions CI/CD

Every push to the main branch triggers a GitHub Actions workflow that builds the Docker image and deploys it to Azure. The entire pipeline — from git push to live deployment — takes under 3 minutes. No manual steps required.

What This Project Demonstrates

URLZAP is deliberately simple as an application — the complexity is in the infrastructure. It demonstrates a complete DevOps loop: containerized application, cloud hosting, and automated CI/CD. The same pipeline pattern applies to far more complex services — which is exactly why I built it this way.

If you want to see the deployment pipeline in detail, I wrote a full walkthrough in the Azure Deployment blog post.
