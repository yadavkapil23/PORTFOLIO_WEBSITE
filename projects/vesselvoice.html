VesselVoice

An underwater acoustic classification system — a multi-class audio pipeline using CNN and ResNet-18 that classifies vessel sounds across 5 categories using spectrogram analysis and a real-time FastAPI backend.

The Problem

Identifying vessels from underwater acoustic signals is a critical challenge in maritime surveillance, environmental monitoring, and naval applications. Traditional approaches rely on manual expert analysis — slow, expensive, and not scalable. VesselVoice automates this using deep learning on audio spectrograms, achieving real-time classification across 5 vessel categories.

Audio Classification Pipeline

Raw underwater audio is converted into mel spectrograms — 2D visual representations of sound frequency over time. This transforms the audio classification problem into an image classification problem, enabling the use of powerful CNN architectures like ResNet-18 that are proven on visual data.

import librosa
import numpy as np

def extract_spectrogram(audio_path: str) -> np.ndarray:
    y, sr = librosa.load(audio_path, sr=22050)
    mel_spec = librosa.feature.melspectrogram(
        y=y, sr=sr, n_mels=128, fmax=8000
    )
    mel_db = librosa.power_to_db(mel_spec, ref=np.max)
    return mel_db  # shape: (128, time_frames)

ResNet-18 for Spectrogram Classification

ResNet-18 was chosen for its residual connections that prevent vanishing gradients in deeper layers, making it highly effective for extracting hierarchical features from spectrograms. The model was fine-tuned on underwater acoustic data, classifying signals into 5 vessel categories with strong accuracy across all classes.

Custom Feature Extraction

Beyond mel spectrograms, the pipeline also extracts MFCCs (Mel-Frequency Cepstral Coefficients) and spectral features — centroid, bandwidth, rolloff, and zero-crossing rate. These handcrafted features complement the deep features learned by ResNet, improving robustness on noisy underwater recordings.

def extract_features(y, sr):
    mfccs = librosa.feature.mfcc(y=y, sr=sr, n_mfcc=40)
    centroid = librosa.feature.spectral_centroid(y=y, sr=sr)
    bandwidth = librosa.feature.spectral_bandwidth(y=y, sr=sr)
    rolloff = librosa.feature.spectral_rolloff(y=y, sr=sr)
    zcr = librosa.feature.zero_crossing_rate(y)
    return np.concatenate([
        mfccs.mean(axis=1),
        centroid.mean(axis=1),
        bandwidth.mean(axis=1),
        rolloff.mean(axis=1),
        zcr.mean(axis=1)
    ])

Real-Time Inference Engine

The inference engine uses sliding window segmentation — the audio stream is divided into overlapping windows, each classified independently. The final prediction is determined by probability averaging across all windows, making the system robust against momentary noise or signal dropout that would fool a single-pass classifier.

def sliding_window_inference(audio, window_size=2.0, hop=0.5, sr=22050):
    predictions = []
    start = 0
    while start + window_size * sr <= len(audio):

FastAPI Backend

The classification pipeline is served via a FastAPI REST endpoint. Audio files are uploaded, processed through the full pipeline, and a prediction with confidence score is returned in JSON. The async design ensures the server can handle multiple concurrent requests without blocking.

React Dashboard

A React frontend provides a clean interface for uploading audio files, visualizing the spectrogram in real-time, and displaying classification results with confidence bars per category. The dashboard communicates with the FastAPI backend over REST.

Docker + CI/CD Deployment

The entire application — FastAPI backend and React frontend — is containerized with Docker and deployed on Render with an automated CI/CD pipeline. Every push to the main branch triggers a fresh build and deployment, ensuring the live version is always up to date.

Results

The system achieves reliable multi-class classification across 5 vessel categories using spectrogram analysis. The sliding window inference with probability averaging significantly improves prediction stability on real-world noisy underwater recordings compared to single-window classification.
